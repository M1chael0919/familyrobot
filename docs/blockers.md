# 阻塞项

## 2026-06-03：启动阶段第一句唤醒词可能漏识别

- 现象：窗口刚打开时，可以听到视频声音但是不会触发交互。后面听不到视频声音，但是交互正常。
- 影响：唤醒词流程在启动瞬间可能漏掉第一次输入。听不到视频声音但是不影响交互。
- 当前状态：已记录，暂时跳过。
- 临时规避：等监听稳定后再进行场景识别。
- 后续处理：
  - 把语音监听、视频解码、TTS 播放拆成独立的启动阶段，避免互相抢占初始化时机。
  - 在 GUI 启动后预热一小段监听窗口，再开始正式唤醒判断。
  - 如果后续仍然漏识别，优先检查视频音频提取、ASR 首轮缓冲和主线程阻塞，而不是直接调高唤醒词阈值。
  - 这个问题保留为“启动时序问题”，后续如果重构音视频管线，再一起做统一复盘。
