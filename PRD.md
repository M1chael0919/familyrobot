# Family Robot PRD

## 1. 产品概述

Family Robot 是一个面向家庭场景的本地交互机器人 Demo。它不是单纯的摄像头识别程序，而是一个“能看见人、知道是谁、会简单回应”的家庭陪伴原型。

典型场景是：机器人放在客厅、餐厅、玄关或书房，摄像头前会反复出现爸爸、妈妈、孩子、老人和访客。系统需要先识别出画面中的人，再持续跟踪这个人，最后把临时轨迹映射到长期身份上。这样即使同一个人离开画面、换了姿势、戴上眼镜或口罩，再回来时，系统依然尽量认出他还是同一个家庭成员。机器人的人机交互这一方面也需要考虑，如果识别出来就问候，会触发多次，在这个demo中我设计了唤醒词触发交互，机器人在听到“你好”之后会用tts把问候语用语音方式给出，tts的接入也方便后期llm接入机器人实现更深层的人机交互。

这个项目的核心不是“识别画面里有个人”本身，而是回答家庭场景里真正有价值的问题：

- 这是谁
- 他是不是之前登记过的家庭成员
- 他离开一段时间后回来，换个穿搭，换个造型，面部部分遮挡时身份是否还能保持一致
- 机器人该不该在合适的时候主动问候
- 语音唤醒后，应该把回应发给谁
- 多人场景中定位是谁唤醒的机器人

因此，这个 Demo 更接近“家庭陪伴 + 家庭看护 + 轻交互”的产品原型，而不是纯视频分析工具。

## 2. 目标

- 本地检测画面中的人
- 本地持续跟踪多个人
- 建立长期身份映射
- 输出个性化问候
- 支持本地语音唤醒与简单交互
- 支持实时摄像头和本地视频测试
- 支持可视化 GUI
- 支持多人场景中的交互人定位
## 3. 典型使用场景

### 3.1 家庭日常场景

机器人开机后放在客厅，摄像头对着沙发和电视区。爸爸经过时，系统识别到他是已登记成员，显示对应身份标签，并在触发唤醒词的时候输出问候。妈妈随后进入画面，系统能分别维持两个人的轨迹和身份，不会把两个人混成一个。

### 3.2 人离开后再回来

爸爸离开画面十几秒，DeepSORT 的 Track ID 可能变化，但系统的身份层应尽量恢复到“爸爸”这个长期身份，而不是把他当成一个全新的人。

### 3.3 轻遮挡和外观变化

当人戴眼镜、摘眼镜、戴口罩、侧脸、半遮挡时，系统不追求绝对准确，但应尽量保持较好的召回，避免频繁把熟人识别成陌生人。

### 3.4 唤醒式交互

用户说“你好”后，机器人再根据当前识别到的身份输出对应问候，例如：

- 爸爸 -> “爸爸今天工作怎么样”
- 妈妈 -> “妈妈今天工作怎么样”
- 孩子 -> “今天作业写完了吗”
- 爷爷 -> “爷爷要不要喝口水”
- 奶奶 -> “奶奶要不要给您扇扇子”


### 3.5 多人同时在场时的发起者识别

在一家五口同时出现在画面里时，用户说“你好”后，系统实际上很难只靠单个普通麦克风精确判断到底是谁先开口。  
正式方案上，可以考虑：

- 麦克风阵列
- 声源定位
- 活跃说话人检测
- 音视频时间对齐
- 唇动辅助判断

但当前项目阶段不实现这些正式能力，先采用 mock 方式。这里的 mock 只针对“多人场景中的发起者识别”，不是整条语音交互链路：

- 根据当前视觉身份状态推测发起者
- 优先选择最近、最稳定、置信度较高的已识别成员
- 如果多人同时在场，则明确标记为“推测结果”

这样可以保证演示流程成立，但不会把 mock 结果误当成真实说话人识别。

### 3.6 真实场景的鲁棒性思考

- 多人遮挡时，优先维持已有身份的连续性，而不是频繁重置成陌生人。`track_id` 的变化不应破坏 `person` 框对应的身份一致性。
- 同一个人换衣服、转身、走出画面再回来时，`Track ID` 可以变，但长期身份应尽量保持不变，这一点通过身份层注册来实现。
- 光线不足、画面模糊、侧脸或半遮挡时，允许身份置信度下降，但不应轻易把已知成员直接改判成其他成员。人脸登记时可以增加样本数量和样本质量。在 `demo.mp4` 中，大约 1 分 50 秒处，遮住下半张脸、走出画面再回来时仍然能识别出来。
- 如果识别不够稳定，问候语宁可退回成通用提示，也不要强行输出错误的个性化内容。在本系统中，如果识别置信度低，会显示“未识别”，但听到唤醒词“你好”时，仍会返回“我在”。
- 多人场景中，如果不能确定是谁唤醒了机器人，需要做发起人定位。
- 边走边说时，语音唤醒可能先于视觉身份稳定，因此交互层应支持“先触发、后补全身份”的流程。
- 在遮挡场景下，person 框重叠时可能会出现多个脸一起进入 InsightFace 的情况，导致识别精度下降（比如爸爸和儿子站在一起时，爸爸的 person 框里包含了儿子的脸）。解决方法是裁剪 person 框，取上端 1/3，更贴合头部，尽量让输入给 InsightFace 的区域只包含单个人脸。

## 4. 产品边界

### 4.1 需要做的

- 人体检测
- 多目标跟踪
- 本地人脸登记
- 身份匹配
- ReID 恢复
- 问候输出
- 唤醒词 + ASR + TTS
- 实时 GUI

### 4.2 核心技术点

这部分是本项目真正主打的能力，不只是“能跑起来”，而是能把家庭场景里的关键链路串起来：

- YOLO + DeepSORT 的视觉检测与跟踪链路
- 轨迹 ID 和长期身份 ID 分离
- 本地人脸登记与身份映射
- 离开再回来后的 ReID 恢复
- 基于身份的个性化问候
- 本地唤醒词、ASR、TTS 的完整语音交互链路
- 实时 GUI 中的框、轨迹、身份和交互提示展示

### 4.3 暂时不做的

- 大模型聊天系统
- RAG 知识库问答
- 云端部署
- 生物识别级安全方案
- 复杂智能家居联动

## 6. 用户流程

1. 用户完成本地登记
2. 机器人启动摄像头或视频输入
3. 系统检测并跟踪画面中的人
4. 系统把轨迹映射到长期身份
5. 当识别到已登记成员时，输出问候
6. 用户通过唤醒词触发语音交互
7. 系统根据当前身份返回个性化回应

## 8. 成功标准

- Demo 可以在本地运行
- 能看到检测框、轨迹 ID 和身份标签
- 熟人可以被稳定映射到长期身份
- 人离开后再回来，身份仍尽量保持一致
- 能输出简短个性化问候

## 9. 关键原则

- Track ID 是临时的
- Identity 是长期的
- 唤醒词只负责触发
- 身份判断必须基于身份层
- 核心能力尽量本地化，便于复现和展示
