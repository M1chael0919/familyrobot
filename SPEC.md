# Family Robot 需求说明

## 1. 目标

实现一个本地家庭交互机器人 Demo，能够检测人物、跟踪人物、保持家庭成员的长期身份，并在识别到熟人时输出简短问候。

## 2. 场景

系统用于家庭场景。摄像头下会出现爸爸、妈妈、孩子、老人等多个家庭成员。某个人离开后再回来，轨迹 ID 可以变化，但身份应该保持不变。

## 3. 功能要求

### 3.1 人物检测

- 检测画面中的人物
- 忽略非人物目标
- 提供边框和置信度

### 3.2 多目标跟踪

- 跟踪多个目标
- 为短期运动连续性维护临时轨迹 ID
- 支持短暂遮挡和离开后的重新进入

### 3.3 身份管理

- 为已登记的家庭成员分配长期身份
- 身份与轨迹 ID 必须分离
- 人物离开后再回来，身份仍然保持一致
- 身份层应支持本地人脸特征提取与相似度检索，推荐采用 `InsightFace + FAISS` 这类本地方案
- 身份登记样本应可复用，用于稳定识别和 ReID 恢复

### 3.4 问候输出

- 对识别到的熟人输出简短问候
- 可以先文本输出，再扩展 TTS

### 3.5 语音交互

- 唤醒词仅作为触发器
- 语音回复应跟随当前视觉身份
- 支持本地 ASR 和 TTS

## 4. 非功能要求

- 全部本地运行，不依赖云服务
- 适合演示，逻辑清晰、可解释
- 允许先做 Demo 级别实现，不追求生产级生物识别安全

## 5. 约束

- 视觉主链路必须使用 `YOLO + DeepSORT`
- 身份层不能直接把轨迹 ID 当作最终身份
- 身份层应在本地完成 embedding 提取和身份匹配，不依赖云端人脸服务
- 语音只能负责唤醒与交互，不替代视觉身份

## 6. 验收标准

- 能在本地摄像头或视频中检测人物
- 能持续跟踪人物
- 能把临时轨迹映射为稳定身份
- 能在本地完成身份样本登记、特征提取和身份匹配
- 同一人离开再回来后，身份不应漂移
- 能输出简短个性化问候
- 识别需要流畅
